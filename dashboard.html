A null pointer dereference happens when the program reads/writes through a NULL pointer (address 0), causing an immediate segmentation fault. One of the most common crash bugs found by fuzzers.

Description

Integer overflow occurs when arithmetic produces a value exceeding the type's max. For 32-bit int, 2,147,483,647 + 1 wraps to -2,147,483,648. Can cause logic bypasses, corrupt allocations, and security vulnerabilities.

Fix: Validate ranges. Use long long. Compile with -fsanitize=integer to detect at runtime.
